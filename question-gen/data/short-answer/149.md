# Background

Consider the following code segment:

string bigOh2(int n)
    if (n == 0) return "a";
    string str = bigOh2(n-1);
    return str + str;

# Parts

## a

Find a lower bound (Big Omega) on the best case runtime. Then determine if this is a tight lower bound (Big Theta).

**Answer:** The function calls itself $n$ times, so one might guess $\Omega(n)$. However, the string concatenation cost matters: at level $k$, the output string has length $2^k$. The concatenation str + str at level $k$ produces a string of length $2^k$, costing $O(2^k)$. Total cost: $\sum_{k=0}^{n} 2^k = 2^{n+1} - 1$. So $\Omega(n)$ is not tight. The tight bound is $\Theta(2^n)$.
