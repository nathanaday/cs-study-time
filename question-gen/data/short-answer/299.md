# Background

A computer performs $10^{10}$ operations per second, and you need results within one hour ($3.6 \times 10^{13}$ operations).

# Parts

## a

What is the largest input size $n$ for an $O(n^2)$ algorithm?

**Answer:** $n \le \sqrt{3.6 \times 10^{13}} \approx 6 \times 10^6$.

## b

What is the largest input size $n$ for an $O(2^n)$ algorithm?

**Answer:** $2^n \le 3.6 \times 10^{13}$, so $n \le \log_2(3.6 \times 10^{13}) \approx 45$.

## c

What is the largest input size $n$ for an $O(2^{2^n})$ algorithm?

**Answer:** $2^{2^n} \le 3.6 \times 10^{13}$, so $2^n \le \log_2(3.6 \times 10^{13}) \approx 45$, giving $n \le \log_2(45) \approx 5$.
